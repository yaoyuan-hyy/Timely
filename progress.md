# Timely 项目进度

更新时间：2026-06-11

## 当前 MVP 目标

Timely 当前阶段只聚焦“记录”：帮助用户用自然语言记录过去或未来某个时间点发生的事情，并在日历中查看、修改和清除这些记录。

暂不扩展提醒、复杂规划、多人协作、账目统计等能力。后续如果加入账目记录，也应沿用“自然语言输入 -> 结构化记录 -> 日历/列表展示”的主路径。

## 已完成

### 1. 产品范围收敛

- 已将 MVP 核心目标收敛为“记录事件”。
- 保留事件记录所需的数据结构：事件、对话消息、待补充信息。
- 提醒相关能力暂时不作为主流程。

### 2. 首页对话 UI

- 首页改为淡蓝色与白色为主的移动端界面。
- 左上角菜单按钮可打开侧边栏。
- 侧边栏可切换三个主要界面：对话、记录、设置。
- 底部输入区改为输入栏 + 麦克风按钮。
- 对话区支持用户消息和系统回复展示。

### 3. 记录日历 UI

- 记录页改为日历形式展示。
- 年份和月份可点击选择。
- 日历支持上下滑动浏览临近月份。
- 点击某一天后进入单日事件时间轴。
- 单日视图中可查看当天记录，并可取消对应事件。
- 整体字体和布局已按移动端观感调小、压紧。

### 4. MiniMax-M3 接入

- 已预制并接入 OpenAI SDK 兼容风格的 MiniMax-M3 API 配置。
- 支持环境变量：
  - `OPENAI_BASE_URL`
  - `OPENAI_API_KEY`
  - `OPENAI_MODEL`
- 默认 base URL 为 `https://api.minimaxi.com/v1`。
- API route 为 `/api/record-event`。

### 5. 自然语言创建事件

- 用户可以输入类似“帮我记录一下6月13日下午3点开会”。
- AI 会解析为结构化事件。
- 前端会把结构化结果写入本地事件状态。
- 如果 AI 返回不完整或请求失败，会回退到本地解析逻辑。
- 支持缺少时间时追问“什么时候？”。

### 6. 自然语言删除事件

- 已支持删除/取消/清除已有事件。
- 用户可以输入：
  - “帮我删除6月13日的会议”
  - “把这周六的会议给我删除了”
- MiniMax schema 已支持 `delete_event` 和 `targetDate`。
- 删除逻辑不会物理移除事件，而是将事件状态标记为 `cancelled`。
- 日历只展示 active 事件，因此被删除的事件会从日历中消失。
- 如果找不到匹配记录，会回复“没找到这条记录。”。
- 如果同一天有多条同名记录，会追问具体时间。

### 7. 静态预览页同步

- `public/app.js` 已同步接入 `/api/record-event`。
- 静态预览也支持创建事件和删除事件。
- API 不可用时会回退到本地解析。

## 当前已知注意点

- 事件数据目前仍存储在浏览器本地状态/localStorage 中。
- 删除事件只是标记为 `cancelled`，不是从数据中彻底删除。
- 多条相似记录的删除能力已做基础追问，但后续还可以继续优化多轮消歧体验。
- 麦克风按钮目前是 UI 状态入口，尚未接入真实语音识别。
- `.env.local` 用于本地真实 API 测试，不应提交。

## 建议下一步

1. 优化删除事件的多轮追问体验，例如用户说“下午那个”时更自然地匹配下午事件。
2. 在日历单日视图中增加“已取消记录”的查看入口，方便恢复或确认历史。
3. 接入真实语音输入，让麦克风按钮从 UI 状态变成可用能力。
4. 将本地事件状态抽象为可替换的数据层，为后续接数据库做准备。
5. 如果开始账目记录，需要新增独立 record type，避免和日程事件混在同一结构里。
